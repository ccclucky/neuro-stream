## ✨ 核心痛点与解决方案 (Why NeuroStream?)

在当前的 AI Agent 生态中，智能体极具行动力，却**跑不通“花钱买服务”的最后一公里**：
- 传统 Web2 支付（如 Stripe/信用卡）对机器极不友好，必须有人类介入。
- Agent 直接转账给 Web2 提供商存在巨大的“信任黑洞”——付了钱，API 突然挂了或不返回结果怎么办？

**我们的终极解法：Protocol (协议) + Gateway (网关) 的双核架构。**

**1. 重新定义机器间的支付信任 (基于 Escrow 的担保机制)**
我们把传统的 API 调用变成了一手交钱一手交货的“原子操作”。Agent 的资金会首先锁定在 Monad 测试网（高性能、低延迟）的智能合约中（Escrow）。
- **保护 Agent**：只有当 Provider 返回了合法的结果并上链验证后，资金才会释放；若超时未交付，全额自动退款。
- **保护 Provider**：看到链上锁定的定金才开始干活，确保“发包即收款”。

**2. 首创 402 Agent Payment Gateway (支付网关中间件)**
完全受 HTTP 402 (Payment Required) 理念启发。网关拦截任何未经授权的 API 调用，返回 402 挑战（Challenge），要求调用方出示链上锁定凭证。这就无缝地把 Web2 的 HTTP 世界和 Web3 的智能合约世界桥接在了一起。

## 🛠️ 项目落地的技术亮点 (How it works?)

很多项目只停留在白皮书阶段，而 **NeuroStream 已经是一套完整可运行的基础设施平台 (Monorepo)**。

我们的技术栈与实现亮点：
- **极简 SDK（零摩擦接入）**：Agent 开发者只需一行代码 `client.callService()`，NeuroStream SDK 就会在后台自动完成：链上鉴权 (`Approve`) -> Escrow 锁资 -> 获取结果，这让 LLM 处理复杂支付逻辑时如丝般顺滑。
- **Provider (提供商) 零区块链门槛**：作为 Web2 服务商，你**完全不需要懂区块链，也不需要私钥钱包**。你只需提供一个普通的 HTTP API，NeuroStream 基础设施（网关和异步事件索引器 Indexer）会全程帮你接管 Web3 的流量并用 USDC 结账给你。
- **去中心化的事件驱动架构**：基于 Viem + Supabase 轮询索引器，实现链上状态与链下 PostgreSQL 数据库的毫秒级同步，确保每一笔微支付（Micro-payment）可被追溯、统计与可视化。
